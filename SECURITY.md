# Security Policy

## Supported Versions

Security fixes are prioritized for the latest release on the default branch and
active release-prep branch.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead, report privately using GitHub Security Advisories for this repository
("Report a vulnerability"), or contact the maintainers through the project's
private channel.

When reporting, include:

- A clear description of the issue
- Reproduction steps or proof-of-concept
- Impact assessment
- Suggested remediation (if available)

We will acknowledge reports as quickly as possible and provide updates as we
triage and remediate.

## Disclosure Process

- We validate and reproduce the report.
- We develop and test a fix.
- We coordinate disclosure timing with the reporter.
- We publish a patch release and advisory notes.
